# 决策记录 0002：ASR Provider Go/No-Go

- 状态：已提议
- 日期：2026-07-19
- 关联任务：Linear ZER-106、ZER-107、ZER-108
- 决策范围：M0 可行性与合规门槛

## 背景

项目需要在进入 M1 前明确 ASR Provider 的实现和验证边界。当前候选私有 Provider 缺少已确认的许可证、服务条款许可、书面授权和可审计的官方接入资料，因此不能把技术上可能实现等同于当前有权实施。

本记录是工程风险决策，不替代法务意见。授权或条款发生变化时，必须重新评审并留下可追溯证据。

## 决策

### 当前私有 Provider：NO-GO

在许可证、服务条款和书面授权未确认前，停止以下工作：

- 实现、移植或集成私有 Provider；
- 使用真实凭据、真实设备身份或真实服务进行网络验证；
- 从非官方实现、客户端产物或流量记录复刻协议；
- 将私有 Provider 作为 MVP、生产或分发版本的依赖；
- 用一次成功演示替代稳定性、安全性和合规证据。

该结论同时阻塞 ZER-106、ZER-107 中依赖真实私有服务的验收项。

### 通用离线与 Fake Provider：GO

可以继续推进：

- provider-neutral 的 ASR 契约、会话状态机和错误模型；
- 使用团队自行定义事件的 Fake Provider 与本地回环服务；
- 使用合成 PCM 的音频格式、分帧、背压和资源释放测试；
- 超时、取消、断网模拟、final 去重和幂等关闭测试；
- 通用凭据接口、Android Keystore 封装及脱敏测试；
- 不连接真实私有服务的 Android 录音生命周期验证。

这些成果不得隐含任何特定私有 Provider 的兼容性或可用性结论。

### 官方或书面授权 Provider：CONDITIONAL GO

只有在下列条件全部满足后，才可以创建隔离的真实 Provider 适配器并开展联网测试：

1. 服务方或权利人提供可核验的书面使用与测试授权；
2. 法务确认适用许可证、服务条款、隐私、数据处理和分发边界；
3. 获得官方或经授权的版本化接入文档；
4. 使用合法签发的沙箱、测试租户和最小权限短期凭据；
5. 明确速率限制、数据保留、删除、地域和跨境处理规则；
6. 测试参与者已知情同意，测试设备和账户归团队合法控制；
7. Provider 适配器与通用核心分包隔离，能够停用和替换；
8. 安全审查确认凭据、音频和转写内容不会进入普通日志或提交物。

任一条件缺失时，状态保持 NO-GO。

## Clean-room 边界

允许进入正式代码和测试的内容：

- 团队独立定义的 provider-neutral 接口、状态和错误；
- 公开标准及其公开测试向量；
- 合成音频、人工文本和团队自定义 Fake 事件；
- 官方文档或书面授权资料允许使用的最小必要内容。

实现人员不得把未授权材料直接或间接转换为产品代码。对外部参考的观察记录应与正式实现隔离，并由未接触受限实现材料的人员依据独立规格开发。

## 禁止进入代码、测试和提交物的材料

- 未授权参考源码及其翻译、改写或结构性复刻；
- 私有端点、协议路径、请求头、消息名称、字段和枚举；
- 私有设备身份、客户端指纹、注册或凭据获取流程；
- 私有签名、密钥派生、票据或封包参数；
- 抓包载荷、客户端提取物和反编译产物；
- 真实 Token、Cookie、密钥、设备凭据或账户资料；
- 未脱敏的真实音频、用户转写文本和服务响应；
- 能够还原上述内容的日志、fixture、截图或实验报告。

## 解除 M1 阻塞的证据清单

ZER-108 只有在以下证据可追溯且评审通过后，才能给出 GO，或确认 CONDITIONAL GO 的条件已经满足：

- [ ] provider-neutral 契约、状态机和 P0 离线测试通过；
- [ ] Android 录音、权限、前台服务和目标系统限制有真机证据；
- [ ] 目标 Provider 的许可证、服务条款和书面授权已经归档；
- [ ] 隐私、数据处理、保留、删除和跨境规则已经确认；
- [ ] 合法沙箱下有成功率、延迟分布、连续会话和失败分类数据；
- [ ] 重试上限、超时、熔断、取消和资源释放门槛已经冻结；
- [ ] 支持的 Android 版本和设备范围已经冻结；
- [ ] 凭据存储、日志脱敏和秘密扫描已经通过安全审查；
- [ ] 风险负责人、回滚方案和替代 Provider 路径已经明确；
- [ ] 每个未关闭风险已有接受、缓解、替代或阻塞结论。

缺少授权或条款证据时，不得将决策表述为“无风险”，也不得解除 M1 阻塞。

## 影响

- ZER-106 可先交付通用凭据安全层，但真实注册、握手和凭据复用保持阻塞。
- ZER-107 可先交付离线音频、Fake/回环和失败测试，但真实私有流式链路保持阻塞。
- ZER-108 负责维护证据矩阵并作最终决策。
- UI 和其他 M1 功能不得通过引入未授权 Provider 绕过本门槛。

## 复审触发条件

出现以下任一变化时重新评审本记录：

- 获得新的官方 Provider、书面授权或适用条款；
- Provider 的数据处理、技术接口或分发政策变化；
- 产品从内部验证扩展到外部测试、发布或商业用途；
- 安全、隐私或法务发现新的重大风险。

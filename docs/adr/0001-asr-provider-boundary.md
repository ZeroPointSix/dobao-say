# ADR-0001：ASR Provider 边界

## 状态

已接受。

## 决策

- UI、录音、本地存储和领域层只能依赖 `asr-core` 的通用 API。
- 具体 Provider 只能通过 `AsrDriver` 上报通用信号和错误。
- 私有 DTO、凭据、端点、协议字段和服务常量不得进入核心模块。
- Fake Provider 使用团队自行定义的信号、合成 PCM 和人工文本。
- 真实 Provider 只能基于官方文档或书面授权 API 单独实现。

## 原因

协议实现可能变化，且具有许可、服务条款、隐私和风控风险。通用边界能够隔离这些风险，并让状态机、失败恢复和 UI 在没有真实服务时完成测试。

## 代价

需要维护一层映射，Provider 私有能力不会自动进入通用业务层。新增通用能力必须由明确产品需求证明。
